Для начала работы с моделью памяти, прежде всего необходимо создать объект класса ScPerformer. Этот класс отвечает за все действия по обеспечению работы с базой данных, а также за все основные операции необходимые для работы с сементической сетью. Этот класс является абстрактным, а его реализация находится в классе ScPerformerImpl.

Для обеспечения взаимодействия с базой данных, используется вспомогательный класс DataBaseManager, в котором имеется всё необходимое для создания подключения к базе данных и предоставления возможности по работе с ней (он содержит объект класса AbstractGraphDataBase базы neo4j). Также этот класс отвечает за создание узлов, которые представляют типы SC элементов. (по умолчанию создаёт все стандартные типы, а так же может по необходимости создавать и дополнительные типы)

Как уже говорилось ранее, для начала работы необходимо просто создать объект класса ScPerformer передав в конструктор путь к катологу с базой. Если база ранее ещё не была создана, создание произойдёт автоматически.

ScPerformer performer = new ScPerformerImpl("d:/database");

Класс ScPerformer содержит методы для управления транзакциями, поиска узлов в памяти по их имени, удалению дуг и узлов, получению объекта фабрики, которая обеспечивает генерацию элементов в памяти, а также получению итераторов для обхода элементов семантической сети.

Для обеспечения возможности поиска узлов по имени, создаётся специальный индекс, в котором все узлы, у которых устанавливается имя, автоматически индексируются по свойству _scNodeName.
